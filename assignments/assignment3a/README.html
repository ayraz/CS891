Java ExecutorCompletionService Framework, Java ConcurrentHashMap, and Fair Semaphore (Assignment 3 - Part A)

Objectives and Overview of the Assignment

The objective of Part A of this assignment is to learn how to use the Java ExecutorCompletionService framework. Videos describing this framework appear here and here. I recommend you watch these videos to ensure you understand the Java ExecutorCompletionService. Naturally, we'll cover these topics in class, as well. In this part of the assignment you will use an instance of the Java ExecutorCompletionService class that's associated with a "cached" pool of threads to enhance the fixed-sized thread pool implementation of the PalantiriSimulator app from Assignment 2A. Assignment 3A focuses on creating a cached pool of threads that allow beings to concurrently gaze into a given number of palantiri.

Assignment Structure and Functionality

The PalantiriSimulator app is packaged as a project in Android Studio version 3.3. This app is written in both Kotlin and Java and demonstrates many Android capabilities. For the purposes of Assignment 3A, however, you only need to be aware of the following three directories:

  • app/src/main/java/edu/vandy/simulator/managers/beings/completionService, which contains the skeleton(s) you'll need to fill in, as described below.

  • app/src/test, which is a set of JUnit tests that exercise many PalantiriSimulator features and can be used to help test the functionality you implement for the assignment.

  • app/src/androidTest, which is an Android Studio instrumentation test that runs your app automatically.

You'll also need to integrate your enhanced solution for the reentrantLockHashMapSimpleSemaphore folder from Assignment 2B and use it as implementation of the PalantiriManager for Assignment 3A (make sure to address all my comments from Assignment 2B in your Assignment 3A solution).

To compile this code you need to use the provided Android Studio project. You can run this project by clicking the green "Run 'app'" button in the Android Studio IDE, which should automatically select an Android Emulator to run, assuming you have one created. The application's minimum API is 24, so you will need to select an emulator that uses an API >= 24. If you do not already have one created you can create it by clicking on the "AVD Manager" button in the Android Studio IDE.

Program Description and "TODO" Tasks

You'll need to modify several files containing the skeleton Java code by completing the "TODO - You fill in here" tasks to provide a working solution. DO NOT CHANGE THE OVERALL STRUCTURE OF THE SKELETON - just fill in the "TODO" tasks!!! In particular, you'll need to finish implementing the following "TODO" tasks for this assignment in the completionService folder:

  • ExectorCompletionServiceMgr.java. You'll need to complete the "TODO" tasks in various methods to implement a BeingManager via a cached thread pool using the Java ExecutorService and ExecutorCompletionService.

  • BeingCallable.java. You'll need to complete a couple of methods in this class, which should look very familiar by now.

  • Assignment.java. You'll need to complete the "TODO" to set the sTypes field to either GRADUATE or UNDERGRADUATE, depending on which version of the assignment that you're implementing.

Your app will be considered correct if it (1) passes all the unit and instrumentation tests and (2) all beings successfully complete all their N iterations. A correct simulation should restrict the number of gazing beings to the number of palantiri. In other words, if there are four palantiri in the simulation, then only four beings should ever be ``gazing'' on the screen at a time.

When the simulation is running, the app view will display visual feedback to show palantiri and being states, as well as the progress of their gazing iterations. If your assignment implementation is correct and does not throw any exceptions the app title bar labels will appear green when the simulation completes. However, red title bar labels will be displayed when the simulation completes if the implementation throws an exception or is not implemented correctly.

Concluding Remarks

This part of the assignment is designed to deepen your knowledge of writing concurrent Java programs using a thread pool in conjunction with the Java ExecutorCompletionService, which enhances the Java ExecutorService mechanisms we covered in Assignment 2A. It doesn't require you to program any Android concurrency/communication frameworks (which we'll cover in upcoming assignments).
