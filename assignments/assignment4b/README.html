Android AsyncTask Framework and Synchronizers (Assignment 4 - Part B)

Objectives and Overview of the Assignment

The objectives of Part B of this assignment are to learn how to synchronize Android AsyncTasks via a readers-writer lock (Java StampedLock) and a semaphore (implemented using your solution to assignment 3b). Videos describing Java StampedLocks appear here and here. I recommend you watch these videos to ensure you understand Java StampedLocks. Naturally, this topic is covered in class, as well.

In this part of the assignment you will use a semaphore and a Java StampedLock to enhance the AsyncTask/ThreadPoolExecutor implementation of the PalantiriSimulator app from Assignment 3A. Assignment 4B focuses on the PalantiriManager, which is used to restrict the number of beings from Middle-Earth who can concurrently gaze into a fixed number of palantiri.

Assignment Structure and Functionality

The PalantiriSimulator app is packaged as a project in Android Studio version 3.3. This app is written in both Kotlin and Java and demonstrates many Android capabilities. For the purposes of Assignment 3B, however, you only need to be aware of the following three directories:

  • app/src/main/java/edu/vandy/simulator/managers/palantiri/stampedLockFairSemaphore, which contains the skeleton(s) you'll need to fill in, as described below.

  • app/src/test, which is a set of JUnit tests that exercise many PalantiriSimulator features and can be used to help test the functionality you implement for the assignment.

  • app/src/androidTest, which is an Android Studio instrumentation test that runs your app automatically.

You'll also need to integrate your solution for the asyncTask folder from Assignment 3A into the latest/greatest asyncTask skeleton and use it as implementation of the BeingManager for Assignment 4B.

To compile this code you need to use the provided Android Studio project. You can run this project by clicking the green "Run 'app'" button in the Android Studio IDE, which should automatically select an Android Emulator to run, assuming you have one created. The application's minimum API is 24, so you will need to select an emulator that uses an API >= 24. If you do not already have one created you can create it by clicking on the "AVD Manager" button in the Android Studio IDE.

Program Description and "TODO" Tasks

You'll need to modify several files containing the skeleton Java code by completing the "TODO - You fill in here" tasks to provide a working solution. DO NOT CHANGE THE OVERALL STRUCTURE OF THE SKELETON - just fill in the "TODO" tasks!!! In particular, you'll need to finish implementing the following "TODO" tasks for this assignment in the stampedLockFairSemaphore folder:

  • stampedLockFairSemaphoreMgr.java. You'll need to complete the "TODO" tasks in various methods to implement a PalantiriManager via a fair semaphore, a StampedLock, and a HashMap.

  • FairSemaphore.java. You'll need to complete the "TODO" tasks in various methods to implement a counting semaphore using a built-in Java monitor object.

Your app will be considered correct if it (1) passes all the unit and instrumentation tests and (2) all beings successfully complete all their N iterations. A correct simulation should restrict the number of gazing beings to the number of palantiri. In other words, if there are four palantiri in the simulation, then only four beings should ever be ``gazing'' on the screen at a time.

When the simulation is running, the app view will display visual feedback to show palantiri and being states, as well as the progress of their gazing iterations. If your assignment implementation is correct and does not throw any exceptions the app title bar labels will appear green when the simulation completes. If the implementation throws an exception or is not implemented correctly when the simulation completes the red title bar labels will be displayed when the simulation completes.

Concluding Remarks

This part of the assignment is designed to get you familiar with synchronizing multi-threaded programs using Java 8 StampedLocks, which are more powerful (and sometimes more efficient) than the spin-locks and ReentrantLocks we covered in previous assignments.
